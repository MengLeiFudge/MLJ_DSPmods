# 主要处理

- 分馏塔处理
    + 如果输入了一个非当前配方处理的物品，怎么处理？直接搬运到产物输出？
    + 优化分馏塔输出判断，由1111改为211
- 交互
    + 可以在分馏模式、交互模式中切换
        + 分馏模式下，相当于运行建筑培养配方的分馏塔
        + 交互模式下，每个口都可以输入物品到中心，也可以选择信号，然后从任意口导出物品（另一种四项分流器？）
    + 在交互模式下，所有接口处理速率总和为1/2/4/8？这个曲线待定，到最后处理速率为“瞬时”，三满带堆叠
        + 这个曲线变动需要靠科技或者建筑升级实现（目前倾向于后者）
        + 无法输入GridIndex在界限外的物品，防止取不出来
- 矿物复制
    + 矿物复制效果受矿物科技影响
    + 矿物按储量决定概率
- 点数聚集
    + 流动输出不是平均点数，而是0点或4点
- 量子复制
    + 添加无限科技，减少量子复制的精华消耗
- 分解
    + 可以在拆解模式、沙土模式、地基模式中切换
        + 拆解模式下，将物品按主配方返还材料，不会产出分解精华
            + 这一点可能要对配方做调整，不再返还1.25倍原材料，而是红色配方1.1倍，金色配方1.25倍？
            + 考虑了一下，或者这样改：
            + 如果配方可增产，则至多返还0.8份材料；否则至多返还1份材料
            + 但是，配方突破之后可以选择让主输出变多，这样就可以超过限制
            + 当然，并不一定要配方突破，行星分解塔也可以有这样的加成；或者分解塔添加等级，每级增加1%产物输出？
            + 更为底层考虑的话，就要想分解塔到底是什么意义。沙土模式、地基模式有意义。而拆解模式真的有意义吗？
        + 沙土模式下，分解产物为沙土、分解精华，沙土直接添加到背包
            + 或许可以考虑传送带输出沙土，然后交互 塔收集？毕竟沙土其实也是物品，可以在传送带上运输
        + 地基模式下，分解产物为地基、分解精华
- 中心背包为物品添加点数统计，取出时按照平均点数取出
- 配方调整
    + 修复配方结尾是Recipe的问题（复制时会显示xxxRecipe，考虑直接Patch CommonAPI）
    + 配方突破时，可以自选特殊强化方向
        + 需要讨论是否有这个必要，因为这个需要玩家选择，而配方有几百个
        + 需要考虑实现的方式，例如配方品质突破后可以获得能自由分配的“属性点”？
        + 需要考虑能有哪些可选的改进方向，例如：
            + 可以让主输出中某一项产出数目增加10%（概率不能增加，因为所有主输出概率之和为100%）
            + 可以禁用某一项主输出
            + 可以禁用某一项额外输出
            + ……
    + 根据输入物品的价值，适当提升精华的获取数目
    + 调整原胚产出分馏塔的概率分配，低级原胚不能产出高级分馏塔
- 提高定向原胚的合成价值，这个需要重新平衡分馏原胚和分馏塔的价值，同时确定价值与配方
- 通用核心、增幅芯片的物品介绍要标注主要来源（其他物品也是如此）

    + 造物之柱是戴森球巨构，只能建造一个（类似恒星炮）
    + 造物之柱材料为太阳帆&造物火箭，造物火箭需要消耗所有种类的精华
    + 接收使用射线重构站，可以选择一个物品然后直接输出（类似交互塔的交互模式）
